# v0.3.2 — Morphology Material Polish

## Mission

Make close-up neurons feel more organic through restrained shader material and
lighting polish, not by adding broad texture systems or blindly increasing
polygon counts.

Current state: the morphology renderer already has soma spheres, tapered tubes,
render-quality overrides, additive blend, bloom compatibility, and a shared
lighting model. The geometry is good enough to start. The missing piece is
subtle surface life and a better active/resting contrast split, without glitter,
banding, or noisy over-bloom.

Done = close-up somas no longer look like plain plastic balls, tendrils have
subtle biological surface variation, identity/region/E-I color modes still work,
and no new asset or public-preset surface is required.

## Product goal

The viewer should feel like they are looking at living neural tissue rather than
simple geometry. The material should improve both still screenshots and active
motion without making the brain noisy or overexposed.

## Context

`architecture/gpu-rendering.md` owns the morphology tube and soma sphere shaders.
`render_morphology.wgsl` already computes lighting from normals, view direction,
and live morphology uniforms. `RenderQualityConfig` already controls tube sides
and soma tessellation, so this plan should not treat polygon count as the main
beauty lever.

`decisions/dev-tooling.md` keeps morphology config on a separate JSON channel
rather than the flat `VisualSettings` Float32Array. If new tunables are needed,
they should go through the morphology config path, not by shifting the frozen
Float32Array contract.

This plan is gated by the roadmap shader/config preflight and shares one
implementation surface with v0.3.3. v0.3.2 remains critical; the coordination
rule is that material and pulse shader/config edits are planned together by one
owner, not that material polish can be skipped. The default goal is to avoid
changing `MorphUniforms`; if review proves new fields are necessary, Rust/WGSL
layout and size asserts must change atomically.

## In scope

- Add subtle deterministic material variation in `render_morphology.wgsl`.
- Apply distinct material treatment for soma spheres and branch tubes.
- Use deterministic inputs only: `neuron_id`, world position, normal, path
  position, `kind`, and existing color mode.
- Preserve existing visual modes: region, E/I, spike-age, voltage-debug,
  activity, identity.
- Preserve additive/no-depth render behavior.
- Keep the first version asset-free: no PNG/JPEG/texture uploads, no sampler
  bind group, no UV unwrap requirement.
- Tune defaults so material is visible close up and subtle from far away.
- Prefer existing lighting/config fields and shader constants. Add dev-panel
  controls only if one or two high-value knobs are needed after review.
- Coordinate any `render_morphology.wgsl`, `MorphUniforms`, or
  `MorphologyConfig` edits with v0.3.3 before code starts.

## Out of scope

- External image textures.
- UV unwrapping or per-vertex UV buffers.
- SDF neuron bodies.
- Mesh subdivision or high-poly body replacement.
- Tube end caps.
- Dendritic spines.
- Selected-neuron microscope mode.
- Public beauty-control UI or public visual presets.

## Material design

### Soma membrane

Soma material should combine a soft base color with low-frequency membrane
variation. Sparse high-frequency speckle is allowed only if it does not glitter
or crawl:

```text
base_color = existing color mode / soma tint
organic = low-frequency noise(normal, world_pos, neuron_id)
speckle = high-frequency noise(world_pos, neuron_id)
rim = existing rim term
active = current spike glow / pulse input
```

Accepted visual traits:

- mottled membrane, not flat fill;
- slight non-uniform brightness across the sphere;
- stronger rim only near active state;
- active core can go bright, but resting soma should stay subtle;
- material variation must not obscure region/identity color;
- no temporal noise, camera crawl, or high-frequency sparkle.

### Branch/tube material

Branches can use path-aware variation so curved tendrils read as biological
filaments, but the effect must stay quieter than pulse motion:

```text
path_striation = noise(path_len * frequency + neuron_seed)
radial_variation = noise(normal * scale + neuron_seed)
kind_tint = dendrite/axon structural tint
active_packet = v0.3.3 will add pulse contribution later
```

Suggested visual traits:

- subtle longitudinal striation along `path_len`;
- slightly different dendrite vs axon material response;
- small neuron-to-neuron color variation;
- no harsh zebra bands;
- no glittering temporal noise.

## Shader approach

Add small deterministic helpers near the existing morphology shader color code:

```wgsl
fn material_hash(seed: u32) -> f32
fn material_noise3(p: vec3<f32>, seed: u32) -> f32
fn soma_material(base: vec3<f32>, n: vec3<f32>, world: vec3<f32>, id: u32) -> vec3<f32>
fn tube_material(base: vec3<f32>, n: vec3<f32>, world: vec3<f32>, path: f32, id: u32, kind: u32) -> vec3<f32>
```

Prefer cheap value/hash noise over expensive multi-octave FBM. This shader runs
for every visible morphology fragment, so the first pass should be conservative.
If the cheap version does not improve screenshots, delete it rather than
escalating into a texture system in this plan.

## Tuning knobs

Start with constants in the shader or fields derived from existing lighting
values. Add JSON morphology-config fields only after visual review proves they
are needed. Do not expose user-facing presets in this plan.

Possible knobs if needed:

```text
material_strength
soma_mottle_strength
branch_striation_strength
speckle_strength
```

If these are added:

- add them to `MorphologyConfig`, not the `VisualSettings` Float32Array;
- extend `MorphUniforms` on Rust and WGSL atomically;
- update layout size asserts;
- classify impact through `SETTING_IMPACT` / morphology descriptor metadata.
- make sure v0.3.3 pulse fields, if any, share the same uniform/config update
  rather than landing as a second independent layout churn.

## Implementation steps

1. Add cheap deterministic material helpers in `render_morphology.wgsl`.
2. Apply soma material in `fs_sphere` after base color selection and before final
   active/resting scaling.
3. Apply tube material in `fs_main` using segment kind, path position, world
   position, normal, and neuron id.
4. Keep output energy bounded so bloom does not turn resting tissue into fog.
5. Compare screenshots with bloom off and bloom on.
6. Tune against identity color and E/I color modes; both must remain useful.
7. Add optional config fields only if hardcoded defaults cannot cover review.
8. If v0.3.3 is ready, land shared uniform/config changes in one coordinated
   shader/config patch.

## Review checklist

- Somas look organic in close-up.
- Tubes have subtle surface life without becoming striped wires.
- Resting morphology remains quiet.
- Active spikes still dominate attention.
- Region/E-I/identity colors remain recognizable.
- Bloom does not wash out the whole brain.
- The shader does not introduce obvious banding, glitter, or camera-dependent
  texture crawl.

## Exit gate

- No new texture asset pipeline is required.
- No public visual preset surface is added.
- No new per-frame allocation or readback is introduced.
- Shared uniform layouts, if changed, have matching Rust/WGSL updates and tests.
- `cargo test -p brain-visualizer` passes.
- `npm run typecheck` passes.
- `npm test` passes.
- `cargo run -p brain-visualizer --example render_check` passes.
- `cargo run -p brain-visualizer --example morph_view` produces accepted close,
  medium, and far frames under
  [`visual-acceptance-contract.md`](visual-acceptance-contract.md).

## Discipline rules

- Do not solve material quality by turning every neuron into high-poly geometry.
- Do not add external texture assets until procedural material has failed visual
  review.
- Do not shift the `VisualSettings` Float32Array indices.
- Do not make material noise time-varying in the first pass.
- Do not make debug/hero visuals the default if they hurt the main view.
- Do not let this plan and v0.3.3 both change `MorphUniforms` without the
  roadmap preflight deciding the shared layout.
- Do not let separate agents independently patch `render_morphology.wgsl` for
  material and pulse work.
