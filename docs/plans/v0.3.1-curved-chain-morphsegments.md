# v0.3.1 — Curved Chain MorphSegments

## Mission

Make tendrils look organic by generating every dendrite/axon branch as a chain
of short curved `MorphSegment`s, while preserving the existing Rust↔WGSL segment
layout and render pipeline.

Current state: morphology already emits branch-only `MorphSegment` records and a
separate soma instance buffer at initialization. The renderer turns each segment
into a shader-generated tapered tube. This is the right contract. The problem is
that the generator still reads too straight and branch angles feel too mechanical.

Implementation note (2026-06-08): the generator-side implementation is complete
in `sim/morphology.rs`. Dendrites now emit sampled curved chains, axon
`path_len` is branch-local instead of cross-sibling, `MorphSegment` remains
48 B, `MorphSphereInstance` remains 32 B, regression tests cover sampled
dendrite counts and parent-endpoint `path_len` continuity, `cargo test -p
brain-visualizer` passes, and `cargo run -p brain-visualizer --example
morph_view` passes. Final visual tuning still depends on Adam review of the
accepted v0.3.0 stage.

Done = individual neurons have flowing dendrite/axon structure, terminal twigs
curve into sockets, branch splits feel like growth rather than random elbows,
`path_len` is accurate along the chain, and no shared layout changes were needed.
Tune this after v0.3.0's procedural brain-shaped stage is visually accepted so
branch density and silhouette are reviewed in the real composition.

## Product goal

A close-up neuron should feel alive even when paused. The viewer should see a
soma with curved dendrite trees and an axon arbor, not a starburst of sticks.
The far view should not become a bright lattice or hairball.

## Context

`architecture/manifold.md` owns morphology generation. `MorphSegment` is the
branch-only 48-byte layout carrying endpoints, radii, `neuron_id`, `path_len`,
`kind`, and `target_id`. Field order is a hard Rust↔WGSL contract and should not
change in this plan.

`architecture/gpu-rendering.md` owns the shader-generated tube draw. It already
receives a flat segment buffer, draws tapered tubes, and uses `path_len` even
though current spike lighting does not use path timing. v0.3.3 will rely on the
`path_len` work from this plan.

`decisions/rendering.md` says morphology supersedes the retired ribbon/cylinder
paths. This plan stays on the morphology path. It does not revive ribbons.

`00-roadmap-index.md` owns the orchestration rule for morphology work: do not
parallelize this plan with shader/config changes that would mutate
`render_morphology.wgsl`, `MorphUniforms`, or `MorphologyConfig` without one
owner coordinating the surface.

## In scope

- Rewrite branch generation helpers to emit curved chains of existing
  `MorphSegment`s.
- Preserve `MorphSegment` and `MorphSphereInstance` layouts exactly.
- Preserve existing `kind` and `target_id` semantics:
  - dendrites remain landing/context structure;
  - shared axon root/cluster segments remain source-owned;
  - terminal twigs continue to carry the real target id.
- Compute cumulative `path_len` along actual emitted chain distance.
- Replace hard-feeling random forks with growth-cone-style branch direction.
- Tune dendrite and axon branch angles separately.
- Update segment-budget math if more chain samples increase per-neuron segment
  count.
- Record expected segment and vertex budgets before tuning the final look.
- Keep morphology deterministic for the same seed, `N`, `K`, and config.
- Keep all-K target coverage unless visual review proves the default is too
  cluttered; if it is cluttered, prefer lowering default K over hiding simulated
  connections.

## Out of scope

- New curve primitive in WGSL.
- Adding Bezier/Catmull-Rom control points to `MorphSegment`.
- Traveling electrical pulses; that is v0.3.3.
- Incoming-target reverse lookup for dendrite orientation.
- Click-to-inspect or selected-neuron rendering.
- Region-specific cell morphology.
- Reviving `DRAW_LEGACY_RIBBONS` or `DRAW_LEGACY_CYLINDERS`.

## Growth model

Build each biological branch as a sampled polyline. The generator emits one
`MorphSegment` between each pair of neighboring samples.

At each step, compute the next direction from weighted influences:

```text
next_dir =
  parent_dir          * direction_continuity
+ outward_from_soma   * outward_bias
+ deterministic_curl  * curl_strength
+ target_pull         * target_attraction
+ split_bias          * branch_angle_bias
```

Then normalize, step forward, taper radius, and emit the segment.

### Dendrite behavior

Dendrites should be shorter, bushier, and more locally curved than axons.
Primary branches should leave the soma with clear separation, then child twigs
should curve gradually rather than splitting at sharp angles.

Suggested behavior:

- primary count stays in the current tuned range unless review demands change;
- primary branches start with strong outward bias from soma;
- child twigs inherit most of the parent direction;
- split angles are moderate and not symmetrical;
- radius taper is smooth and slightly accelerated near terminals.

### Axon behavior

Axon root/cluster branches should feel more directed. They can travel farther,
curve toward target/socket structure, and taper more visibly.

Suggested behavior:

- root starts with strong source-driven direction;
- cluster branches diverge gradually from root direction;
- terminal twigs use target/socket pull more heavily near the end;
- target pull should not create straight chords from soma to target;
- terminal twigs should land near visible dendrite anchors without collapsing
  into one bright knot.

## Parameters

Start with locked defaults in Rust. Expose only the smallest useful subset in
`MorphologyConfig` after review. The dev panel should not become a branch-shape
laboratory unless those controls prove necessary.

Potential generator fields:

```text
curve_samples_primary
curve_samples_twig
curvature_strength
direction_continuity
curl_strength
curl_frequency
target_pull
branch_angle_min
branch_angle_max
```

Protected budget fields remain protected. If the new samples increase segment
counts, update the budget derivation and cap tests rather than exposing cap
sliders.

## Implementation steps

1. Add a small internal `BranchCursor` / `GrowthState` helper in
   `sim/morphology.rs` that tracks current point, direction, radius, cumulative
   path length, and branch kind/target id.
2. Add deterministic curl/noise helpers using the existing hash namespace and new
   morphology-only salts.
3. Rewrite dendrite primary/twig emission through the chain helper.
4. Rewrite shared axon root/cluster emission through the chain helper.
5. Rewrite terminal twig emission through the chain helper and preserve real
   target ids on terminal twigs.
6. Recompute `path_len` from emitted chain distance, not from intended chord
   distance.
7. Update `MorphologyStats` only for facts that review needs; avoid dumping
   recoverable implementation details into docs.
8. Update `max_segs_per_neuron` and cap tests if segment counts change.
9. Record per-neuron segment counts, cap usage, tube vertex count, and sphere
   vertex count in the review artifact JSON.
10. Run `morph_view` with resting structure and active-only frames using the
   shared visual acceptance contract.
11. Tune until close-up and far-view screenshots both pass visual review.

## Review checklist

- Dendrites are visibly curved, not kinked.
- Branch splits are not symmetrical or starburst-like.
- Axon terminal twigs still point toward meaningful target/socket areas.
- The default view does not become an unreadable glowing mesh.
- `path_len` increases monotonically along each emitted chain.
- No segments are silently dropped at default N/K.
- Segment and vertex budgets leave room for v0.3.2 material and v0.3.3 pulse
  shader cost.
- Identity color and E/I tint still read correctly.

## Exit gate

- `MorphSegment` remains 48 bytes and byte-identical with WGSL.
- `MorphSphereInstance` remains 32 bytes and byte-identical with WGSL.
- Morphology generation is deterministic for the same seed/config.
- Segment cap tests pass with no silent truncation at default tier.
- Review artifacts record segment count, cap usage, tube/sphere vertex counts,
  and any dropped-segment count.
- `cargo test -p brain-visualizer` passes.
- `npm run typecheck` passes.
- `npm test` passes.
- `cargo run -p brain-visualizer --example morph_view` produces accepted review
  frames under [`visual-acceptance-contract.md`](visual-acceptance-contract.md).
- `cargo run -p brain-visualizer --example render_check` passes.

## Discipline rules

- Do not change the segment buffer layout to solve curvature.
- Do not allocate or rebuild morphology per frame.
- Do not revive the retired ribbon path.
- Do not make branch shape depend on nondeterministic RNG.
- Do not add public UI controls for every generator knob.
