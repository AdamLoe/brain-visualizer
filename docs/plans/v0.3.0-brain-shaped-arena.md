# v0.3.0 — Procedural Brain-Shaped Arena

## Mission

Make the app read as a brain before improving the micro-geometry. Replace the
sphere-like arena with a recognizable procedural brain-shaped surface and a
cortical-shell-biased neuron placement volume.

Current state: neuron positions are placed inside a unit sphere, while the shell
is a procedurally gyrified icosphere. Gyri/sulci noise adds surface texture, but
the silhouette and neuron cloud still read as a lumpy sphere. This plan keeps
the asset-free procedural manifold direction and makes the outer shell and
neuron cloud agree on one brain-shaped geometry.

Done = from the default camera, top view, and side view, the silhouette reads as
a brain without explanation; the neuron cloud reads as cortical tissue near that
surface instead of filling a glowing solid sphere; all gates pass; and the real
mesh route is explicitly deferred rather than half-implemented.

## Product goal

The first screenshot should not need captions. A visitor should see a brain-like
object first, then notice the neuron activity inside it. If this plan ships well,
all later morphology, material, and impulse work will inherit a believable stage.

## Context

`architecture/manifold.md` owns the generation-time CPU work: manifold surface,
neuron placement, spatial grid, region assignment, and morphology geometry. The
spatial grid is rebuilt from placed positions, so it should adapt automatically
after placement changes. Region assignment is currently hash-shuffled and
position-independent; this plan does not change it.

`architecture/gpu-rendering.md` owns the manifold surface pass. The shader only
needs vertex/index buffers plus the existing surface uniforms. The first version
should keep the current dim-fill shader and change the generated
mesh/placement source, not redesign shell rendering.

`decisions/manifold.md` currently says the manifold is procedural and
asset-free. This plan preserves that decision. Real mesh assets stay in
[`future_roadmap.md`](future_roadmap.md) and are not a fallback route inside
this plan.

## In scope

- Replace the sphere-like silhouette with procedural brain shaping.
- Add hemisphere separation and a visible longitudinal fissure.
- Add anterior-posterior elongation, lobe-like asymmetry, and non-spherical
  profile shaping.
- Keep deterministic gyri/sulci displacement, adjusted to the new envelope.
- Make the generated surface and placement sampler use the same brain-shaped
  envelope.
- Replace full sphere placement with deterministic cortical-shell-biased
  placement plus only sparse interior fill if review needs depth.
- Keep the existing manifold surface shader for the first pass.
- Keep the spatial grid construction unchanged except for consuming the new
  positions.
- Add grid-occupancy and dynamics sanity checks because placement changes can
  affect connectivity even when the connectivity algorithm is unchanged.
- Keep hash-shuffled Input/Association/Output region assignment unchanged.
- Update architecture and decision docs when the code ships.

## Out of scope

- External brain mesh assets.
- Mesh licensing, Blender/MeshLab processing, runtime mesh binary formats, and
  point-in-mesh rejection sampling.
- Glass, fresnel, wireframe, or transparent-shell redesign.
- Spatially coherent region assignment.
- Educational labels or anatomical overlays.
- Runtime old-sphere/new-brain toggle.
- Morphology branch-quality changes; those start in v0.3.1.

## Approach

### Stream 0 — procedural shape grammar

Build a small, deterministic brain-space shaping layer around the existing
surface generation:

- start from the current subdivided sphere direction;
- apply anterior-posterior elongation and side/profile compression;
- split the shape into left/right hemispheres with a visible fissure;
- add lobe-like asymmetry without claiming anatomical atlas accuracy;
- apply gyri/sulci displacement after the envelope still reads as a brain.

The shape function should be a reusable host-side primitive: the surface mesh and
placement sampler must not drift into two different notions of "brain-shaped."

### Stream 1 — cortical-shell-biased placement

Replace pure full-volume sphere placement with deterministic shell-biased
sampling:

1. Draw direction and shell-depth values from the existing deterministic hash
   namespace.
2. Use the same brain-shaped envelope as the surface mesh to find the local
   outer radius.
3. Bias accepted neurons toward a cortical shell near that radius.
4. Add only sparse interior fill if visual review proves the shell looks hollow
   or too thin.
5. Preserve deterministic output for the same seed and `N`.

Do not use `rand`. All candidate draws must come from the repo's deterministic
hash namespace so rebuilds remain reproducible.

### Stream 2 — surface/grid integration

Use the shaped surface vertices/indices as the manifold surface data consumed by
the existing render path. Keep `render_manifold.wgsl` unchanged unless visual
review proves the dim-fill shader cannot show the silhouette.

The spatial grid should continue to rebuild from placed positions. No region or
connectivity rule changes belong in this plan.

Because `target()` starts from each neuron's spatial-grid cell, cortical-shell
placement can still change target locality and empty-cell behavior. Add a narrow
sanity pass after placement:

- grid occupancy has no pathological empty-cell clustering for the default tier;
- target determinism still passes for the same seed and `N`;
- default dynamics do not collapse into silent or saturated behavior after the
  new placement;
- review artifacts include basic spikes/sec and branching-ratio notes when
  available, without treating llvmpipe as a performance benchmark.

### Stream 3 — review harness

Use the shared visual acceptance contract and add/update captures where useful:

- shell-only default view,
- top view,
- side view,
- surface-only view,
- neuron-cloud-only view,
- morphology-on view.

The review question is simple: does the shape read as a brain without the glow
layer doing the work, and does the placement read as cortical tissue rather than
a solid glowing fill?

## Exit gate

- Brain shell is procedural and asset-free.
- Shell-only captures read as a brain from default, top, and side views.
- Neurons are placed in a cortical-shell-biased brain volume, not the old sphere.
- Placement is deterministic for the same seed and neuron count.
- Region ratios still match the existing 30/40/30 intent.
- Spatial-grid occupancy and nearest-occupied behavior remain sane at default
  `N`/`K`.
- Default dynamics smoke does not collapse into silent or saturated behavior
  because of the new placement.
- Real mesh support remains backlog-only in `future_roadmap.md`.
- `cargo test -p brain-visualizer` passes.
- `npm run typecheck` passes.
- `npm test` passes.
- `cargo run -p brain-visualizer --example render_check` passes.
- `cargo run -p brain-visualizer --example morph_view` produces accepted review
  frames under [`visual-acceptance-contract.md`](visual-acceptance-contract.md).

## Discipline rules

- Do not add external assets in this plan.
- Do not reopen a mesh fallback inside this plan.
- Do not add a permanent `use_old_sphere` / `use_new_brain` runtime toggle.
- Do not change `assign_regions` in this plan.
- Do not change the simulation, connectivity rule, or `last_spike` layout.
- Do not add per-frame shape processing; all placement and mesh generation happen
  at initialization/network rebuild time.

## Migration notes

When this ships:

- Rewrite `architecture/manifold.md` surface generation and placement sections
  to describe the procedural brain-shaped envelope and cortical-shell placement.
- Update `decisions/manifold.md` with the accepted decision: procedural
  brain-shaped arena first, real mesh deferred unless needed.
- Update `architecture/gpu-rendering.md` only if shell-rendering semantics or
  surface modes change.
- Keep real mesh asset support in `future_roadmap.md` unless this plan's review
  explicitly reopens that route.
