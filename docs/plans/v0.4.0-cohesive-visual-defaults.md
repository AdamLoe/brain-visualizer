# v0.4.0 — Cohesive Visual Defaults

## Mission

Integrate the v0.3 visual work into one accepted default look. The brain shape,
curved morphology, procedural material, soma pulse, traveling impulses, bloom,
and camera controls should feel like one product rather than independent feature
branches.

Done = the default page opens to a recognizable brain with readable activity,
close-up neurons look organic, pulse motion is legible, performance remains
within the intended tier budgets, the accepted defaults are recorded in
[`accepted-visual-defaults-manifest.md`](accepted-visual-defaults-manifest.md),
and the hidden dev panel exposes useful tuning controls plus review presets.

## Product goal

The default experience should be the best-looking experience, not a dev-tuned
state that only exists on one machine. A visitor should not need to open the dev
panel to get the “good” version, while reviewers still get fast hidden presets
for performance and hero captures.

## Context

v0.3.0 changes the stage. v0.3.1 changes branch geometry. v0.3.2 changes the
material response. v0.3.3 changes spike visual semantics. v0.3.4 changes camera
navigation. This plan is the acceptance/tuning pass that makes those features
coherent.

`decisions/dev-tooling.md` keeps detailed tuning in the hidden dev panel. Public
UI should stay minimal. `decisions/scaling.md` says tier selection is explicit,
not automatic. This plan can tune tier presets and defaults, but it should not
re-arm runtime autoscaling.

The accepted look is coordinated through
[`accepted-visual-defaults-manifest.md`](accepted-visual-defaults-manifest.md).
This plan fills that manifest with exact settings, camera presets, hidden
review-preset payloads, and artifact paths once the look is accepted.

## In scope

- Tune default visual settings after v0.3 features land.
- Tune morphology default generator, material, lighting, pulse, and bloom values.
- Decide which dev-panel controls remain useful and which should be hidden,
  grouped, renamed, or collapsed.
- Add hidden dev-panel review presets for default/performance/hero
  configurations.
- Add review-harness configurations for default/performance/hero comparisons.
- Fill the accepted defaults manifest with the accepted first-load config,
  hidden preset payloads, camera presets, and artifact ledger.
- Ensure defaults work across low/balanced/max tiers.
- Ensure localStorage versioning and reset behavior handle any new settings.
- Add review artifacts that compare default/performance/hero configurations.
- Update docs to describe the accepted default visual pipeline.

## Out of scope

- Education mode.
- Neuron picking or inspect mode.
- Runtime auto-scaling.
- CPU backend revival.
- New simulation model features.
- Major public UI redesign.
- Mesh assets; v0.3.0 is procedural-first and real mesh support stays deferred.
- Public visual preset manager. Hidden dev-panel review presets are in scope.

## Default strategy

The shipped default should optimize for the homepage/demo look while respecting
performance. Review compares these configurations through the harness and hidden
dev panel, but they do not become public modes unless a later product decision
accepts that UI:

| Review config | Purpose | Notes |
|---|---|---|
| default | public experience | best balance; no dev action required; hidden preset must match first-load |
| performance | lower visual cost | reduced morphology density/material/pulse/bloom; hidden review preset |
| hero | screenshots/video | higher tessellation/material/bloom; hidden review preset if costly |

Do not make “hero” the default unless it performs well on the balanced tier.
Do not expose these as public presets in this plan.

## Settings review

### Rendering defaults

Review and tune:

- surface mode and opacity;
- far billboard resting opacity and active boost;
- morphology resting brightness and active boost;
- bloom strength/threshold interaction;
- pulse speed, width, brightness, and decay;
- material strength and soma/tube variation;
- identity/region/E-I color readability;
- render-quality defaults for tube sides and soma sphere tessellation.

### Dev panel controls

After tuning, classify controls into:

- **keep visible**: genuinely useful during visual work;
- **collapse/advanced**: useful but distracting;
- **remove/tombstone**: no longer connected or replaced by better controls;
- **review-config-only**: useful for screenshots or comparison, but not a public
  product control.

Any impact-dot behavior must continue to come from the single metadata source.
Renderer-rebuild controls should clearly remain renderer-rebuild controls.

### Hidden review presets

Add a hidden dev-panel preset workflow if the current dev-panel structure can
hold it without a product UI redesign:

- load accepted default, performance, and hero review presets;
- reset the current profile to the accepted first-load default;
- write preset identity into review artifact JSON;
- keep preset payloads versioned so older localStorage cannot corrupt defaults;
- keep the public page free of preset choices.

The accepted-default preset is the source of truth. It must match the real
first-load defaults, not a separate tuned state.

### Tier behavior

For each tier, verify:

- default N/K is still honest;
- morphology segment caps are safe;
- render-quality overrides do not exceed practical GPU limits;
- low tier still looks intentional, not broken;
- max/hero review config does not introduce layout or buffer-size risks.

## Visual acceptance matrix

Capture and compare:

| View | Checks |
|---|---|
| default camera | brain silhouette, pulse readability, no over-bloom |
| top view | hemispheres/shape readable |
| side view | not spherical, brain-like profile |
| close neuron | soma material, curved tendrils, pulse path |
| active-only | moving impulses readable without resting clutter |
| low tier | still attractive |
| hero/dev | screenshot-ready without changing core semantics |

## Implementation steps

1. Freeze v0.3 feature branches into one integration branch.
2. Reset localStorage and test true first-load defaults.
3. Tune default visual settings from a clean profile.
4. Tune low/balanced/max tier render quality separately.
5. Add hidden dev-panel review presets and keep them out of the public UI.
6. Fill the accepted defaults manifest with exact settings, camera presets, and
   artifact paths.
7. Clean up dev-panel labels/tooltips for controls that remain.
8. Update review harness outputs and artifact JSON if necessary.
9. Run gates and capture final accepted screenshots/video snippets.

## Exit gate

- First-load default looks accepted with no dev panel interaction.
- Low/balanced/max tiers all initialize and render correctly.
- Reset-to-defaults clears new config safely.
- No stale/tombstoned setting changes visible output accidentally.
- Any default/performance/hero variants remain review/dev configurations, not a
  public preset manager.
- Hidden dev-panel presets load accepted default, performance, and hero configs
  and write preset identity into artifact JSON.
- [`accepted-visual-defaults-manifest.md`](accepted-visual-defaults-manifest.md)
  contains the accepted first-load config, hidden preset payloads, camera
  presets, and artifact ledger.
- `cargo test -p brain-visualizer` passes.
- `npm run typecheck` passes.
- `npm test` passes.
- `cargo run -p brain-visualizer --example render_check` passes.
- `cargo run -p brain-visualizer --example morph_view` produces accepted default,
  performance, and hero artifacts if hero review config exists, under
  [`visual-acceptance-contract.md`](visual-acceptance-contract.md).

## Discipline rules

- Do not hide performance problems by turning off meaningful visuals only in the
  default path.
- Do not re-arm automatic scaling.
- Do not add public sliders for developer tuning knobs.
- Do not expose default/performance/hero presets in the public UI.
- Do not let localStorage from old configs corrupt new defaults.
- Do not document llvmpipe numbers as real GPU benchmarks.
