# v0.3.3 — Soma Pulse & Traveling Impulses

## Mission

Make neural activity read as physical motion: when a neuron fires, the soma
pulses once brightly and a visible impulse travels across the curved morphology
segments.

Current state: spike lighting is whole-connection and instant. A firing neuron
lights its structure/downstream twigs and fades with the same glow curve as the
far billboard. `path_len` exists on `MorphSegment`, but it no longer drives
render timing. v0.3.1 makes `path_len` meaningful again by emitting curved
segment chains.

Done = firing starts with a bright soma event, then a compact pulse band moves
along the small curved segments; whole-branch instant lighting is reduced or
removed; the effect is visual-only and does not change simulation dynamics.

## Product goal

A visitor should visually understand the causal event without reading a label:
**cell fires → soma flashes → signal moves through tendrils → downstream area
reacts.** This is the main “alive” feature for the next visual phase.

## Context

`architecture/data-model.md` owns `last_spike`: the packed word stores the
has-spiked bit, neuron type, and last-fire tick. Render shaders already use this
word to compute spike age and glow.

`architecture/gpu-rendering.md` owns the far billboard pass and morphology pass.
The soma pulse should affect both the billboard body visual and the morphology
soma sphere path. The traveling impulse should live in the morphology tube
shader, using `MorphSegment.path_len` plus the local position along the segment.

`decisions/rendering.md` currently says there is no traveling pulse and whole-
connection spike lighting is τ-synced. This plan intentionally revisits that
choice for downstream/source-owned lighting only. It does not reintroduce
misleading upstream lighting.

This plan is gated by the roadmap shader/config preflight in
[`00-roadmap-index.md`](00-roadmap-index.md) and shares one implementation
surface with v0.3.2. v0.3.3 remains a separate acceptance target, but pulse and
material shader/config edits should be planned by one owner before either plan
mutates `render_morphology.wgsl`, `MorphUniforms`, or `MorphologyConfig`. The
expected path is to keep `MorphSegment` unchanged, derive timing from existing
`last_spike`, derive pulse position from v0.3.1 `path_len`, and avoid changing
`MorphUniforms` unless review proves fixed defaults are insufficient.

## In scope

- Add a short, bright soma/body pulse keyed from `last_spike`.
- Add a radius/scale component to the soma pulse where available.
- Add a traveling pulse band in the morphology tube shader.
- Use `path_len` and interpolated per-fragment segment position to place the
  moving pulse.
- Keep the effect visual-only; do not add sim-accurate conduction delay.
- Keep pulse parameters in morphology/render config, not in the simulation
  model.
- Preserve source-owned semantics: axon root/cluster/terminal segments pulse
  from the source neuron's last spike.
- Give dendrites a subtler soma echo only if full outgoing dendrite travel reads
  biologically misleading in review.
- Keep all work on the GPU; no readback, no CPU particle/event list.
- Coordinate all morphology shader/config edits with v0.3.2 material polish
  before code starts.

## Out of scope

- Delay ring buffer or delayed synaptic delivery.
- STDP, plasticity, or learning.
- Incoming/upstream whole-path lighting.
- Retired ribbon renderer.
- CPU backend feature parity.
- Click-to-fire or selected-neuron debug tooling.
- New render pass for pulse particles.

## Visual design

### Soma pulse

A spike should produce a short, punchy event:

```text
age = tick_diff(current_tick, spike_tick)
flash = fast decay over first few ticks
halo = slower decay over following ticks
radius_boost = small decay-scaled scale increase
white_core = optional high-intensity center at very young age
```

The far billboard and the morphology soma sphere should agree on timing. The far
billboard gives the global body flash. The soma sphere gives the close-up
anatomical pulse when morphology is visible.

Suggested parameters:

```text
soma_pulse_brightness
soma_pulse_radius_boost
soma_pulse_tau
soma_pulse_core_ticks
```

### Traveling impulse

Each tube fragment knows an approximate path position:

```text
segment_start = seg.path_len
segment_end = seg.path_len + length(seg.b - seg.a)
local_path = mix(segment_start, segment_end, segment_t)
```

A firing source produces a traveling distance:

```text
age_ticks = tick_diff(current_tick, source_spike_tick)
travel_dist = age_ticks * impulse_speed
pulse = exp(-pow(local_path - travel_dist, 2.0) / pulse_width)
```

The result is a moving band that crosses each short segment and appears
continuous across the chain because v0.3.1 made `path_len` cumulative.

Suggested parameters:

```text
impulse_speed
impulse_width
impulse_decay
impulse_tail_strength
axon_impulse_strength
dendrite_echo_strength
```

### Branch semantics

First pass semantics:

- **Soma** pulses for every firing neuron.
- **Axon/root/cluster/terminal twigs** carry the traveling outgoing impulse from
  the source neuron's `last_spike`.
- **Dendrites** either get a brief local soma echo or a much weaker inward-style
  effect. They should not imply that source-owned dendrites are real outgoing
  synaptic paths.
- **Targets** are not force-lit by the source pulse. They light when the actual
  simulation makes them fire.

This keeps the visual honest while still making motion visible.

## Shader approach

### Tube vertex output

Extend the tube vertex output with a path-position varying:

```wgsl
out.path_pos = seg.path_len + ring_t * length(seg.b - seg.a);
out.seg_kind = seg.kind;
out.neuron_id = seg.neuron_id;
```

`ring_t` is 0 at endpoint `a` and 1 at endpoint `b`; interpolation gives the
fragment-stage local path position.

### Fragment brightness

Replace or blend down the old whole-connection spike boost:

```wgsl
base_resting = existing material/resting brightness
whole_glow = old instant glow * legacy_whole_glow_strength
packet = impulse_packet(path_pos, age, params)
brightness = base_resting + whole_glow + packet
```

Start with `legacy_whole_glow_strength` low or zero. The pulse packet should be
the main event.

### Soma sphere / far body

Apply the same pulse age curve to:

- `render_far.wgsl` billboard radius and brightness;
- `render_morphology.wgsl → vs_sphere / fs_sphere` sphere radius and brightness.

If sphere radius is scaled in the vertex stage, keep draw counts unchanged and
only move vertices. Do not rebuild geometry per frame.

## Config path

Pulse controls belong with morphology/rendering config. Avoid adding slots to the
flat `VisualSettings` Float32Array unless there is no alternative.

If adding fields to `MorphUniforms`:

- update Rust and WGSL layouts atomically;
- update size asserts;
- update `architecture/gpu-rendering.md` layout notes when the plan ships;
- keep defaults safe when older localStorage morphology config is loaded.
- include any v0.3.2 material fields in the same shared layout/config update
  rather than landing two independent layout migrations.

## Implementation steps

1. Complete the shader/config preflight and confirm v0.3.1 has monotonic,
   accurate `path_len` along curved chains.
2. Coordinate with v0.3.2 and decide the combined material/pulse uniform/config
   shape.
3. Add pulse parameters to morphology config or fixed defaults.
4. Add soma pulse curve helper shared conceptually between far and morphology
   shaders.
5. Apply soma pulse to far billboard brightness/radius.
6. Apply soma pulse to morphology soma sphere brightness/radius.
7. Add path-position varying to tube shader.
8. Add impulse packet math to tube fragment shader.
9. Reduce/remove old whole-connection instant brightness.
10. Tune speed/width so the packet visibly crosses multiple short segments.
11. Capture review frames/video with slow speed presets under the shared visual
    acceptance contract.

## Review checklist

- A single firing event is legible as a soma flash first.
- Traveling impulse moves along curved segments instead of lighting the whole
  arbor instantly.
- Pulse speed feels visible at normal and slow simulation speeds.
- Dendrite behavior does not look like false outgoing signaling.
- Bloom enhances the pulse without saturating the entire brain.
- Old whole-connection glow does not drown out the moving packet.
- The effect works with region, E/I, and identity color modes.

## Exit gate

- No simulation dynamics changed.
- No CPU-side per-spike visual event list added.
- No CPU↔GPU readback added.
- `last_spike` packing unchanged.
- `MorphSegment` layout unchanged.
- Any `MorphUniforms` changes are byte-identical in Rust/WGSL and covered by
  asserts.
- `cargo test -p brain-visualizer` passes.
- `npm run typecheck` passes.
- `npm test` passes.
- `cargo run -p brain-visualizer --example render_check` passes.
- `cargo run -p brain-visualizer --example morph_view` produces accepted active
  pulse review artifacts under
  [`visual-acceptance-contract.md`](visual-acceptance-contract.md).

## Discipline rules

- Do not revive ribbons as the main signal visual.
- Do not pretend this is sim-accurate conduction delay.
- Do not add per-frame allocations for pulse state.
- Do not make upstream shared-arbor lighting look like real incoming causality.
- Do not change packed `last_spike` to add pulse data.
- Do not let this plan and v0.3.2 both change `MorphUniforms` without the
  roadmap preflight deciding the shared layout.
- Do not let separate agents independently patch `render_morphology.wgsl` for
  pulse and material work.
